# 前端接口调用文档

## 一、User接口
### 1、登陆
- *调用路径`/api/user/login`*
- *调用格式：*
```http request
POST https://mushu.onlion/api/user/login HTTP/1.1

Content-Type: application/json

{
    id:xx
    password:xx
}
```
- *传入参数：* 
  - 1、id，类型 **int** ,用户id即学号
  - 2、password，类型 **String** ，用户输入的密码明文传递
- *返回格式：*

```json
{
  "id":0,
  "username":null,
  "access": 0,
  "code":null,
  "jwt":null
}
```
- *返回参数*
  - 1、id，类型 **int** ,用户id即学号
  - 2、username，类型 **String** ，用户名
  - 3、code，类型 **int** ，返回代码
  - 4、access 类型 **int** ,用户权限
  - 5、jwt，类型 **String** ，jwt令牌
    
| 返回代码 | 说明        | 
|------|-----------|
| 200  | 登陆成功，有jwt |
| 201  | 登录失败，无jwt |

### 6、上传人物头像

- *调用路径`/api/image/postImage`*
- *调用格式：*
```http request
POST https://mushu.onlion/api/image/postImage HTTP/1.1
Authorization: Bearer jwt
Content-Disposition: form-data; name="id"

id

Content-Disposition: form-data; name="image"; filename="test.jpg"
Content-Type: image/jpeg

image


```
- *传入参数：*
  - 1、id，类型 **int** ,用户id即学号,表单名必须填 **"id"**
  - 2、image 类型为二进制字节流，为图片信息，表单名必须填 **image**
  - 3、Authorization:jwt令牌，传入格式为"Bearer jwt"
- *返回格式：*

```json
 "path"
```
- *返回参数*
- 1、path，类型 **String** ,图片静态地址

## 二、Goods接口

### 1、上传商品信息

- *调用路径`/api/goods/uploadGoodInf`*
- *调用格式：*
```http request
POST https://mushu.onlion/api/goods/uploadGoodInf HTTP/1.1
Authorization: Bearer jwt
Content-Type: application/json

{
  "userId": 0,
  "name": "水瓶",
  "type": 0,
  "price": 2.2,
  "text": "一个很好的水瓶",
  "state": false
}
```
- *传入参数：*
  - 1、userId，类型 **int** ,用户id即学号
  - 2、name，类型 **String** ,商品名字
  - 3、type，类型 **int** ,商品的类别
  - 4、price,类型 **double** ,商品价格
  - 5、text, 类型 **String** ,商品描述
  - 6、state, 类型 **boolean* ,商品上架状态
  - 7、Authorization:jwt令牌，传入格式为"Bearer jwt"


| 商品类别 | 说明     |
|------|--------|
| 0    | 教材书本 |
| 1    | 电子产品  |
| 2    |    服饰生活用品    |
| 3    | 运动器材       |
| 4    | 宿舍家具       |
| 5    | 美妆文具       |

- *返回格式：*

```json
{
  "userID": 0,
  "goodID": ""
}
```
- *返回参数*
- 1、userID，类型 **int** ,用户id即学号
- 2、goodID，类型 **String** ,商品的UUID
