# 前端综合实践课程设计任务书

## 一、校园二手商品交易系统（难度：▲▲▲▲）

### 1. 用户管理
- **用户注册与登录**：在校学生、教职工可通过学号/工号完成注册登录，可找回账号密码。
- **用户信息管理**：用户查看、编辑个人资料，包含姓名、学院、年级、联系方式、校园认证信息，支持头像、简介修改。
- **用户权限管理**：划分普通学生、教职工、平台审核员、系统管理员权限，普通用户仅可发布、购买商品，审核员负责商品与留言审核，管理员管理全平台数据，隔离权限保障校园信息安全。

### 2. 商品档案管理
涵盖二手商品基础信息、实物实拍图、商品描述、新旧程度、购买凭证截图、分类标签归档；完整留存每件商品发布记录，方便审核溯源，减少虚假、违规商品流通。

### 3. 交易发布管理
包含商品发布登记、商品分类筛选、闲置上架、下架回收、交易预约登记、线下面交登记、订单台账统计；统一管理商品上架、下架、成交全流程信息，支持一键编辑商品信息。

### 4. 分类专区管理
实现校园二手资源分区管控，包含教材书本、电子产品、服饰生活用品、运动器材、宿舍家具、美妆文具等分栏设置；支持置顶商品、急出专区、求购板块发布，实时展示商品在售状态。

### 5. 交易管理
包含商品定价设置、线下自提无线上资金、交易明细查询、订单记录等。

### 6. 统计报表
包含用户信息统计、商品发布统计、成交订单统计、热门品类统计；表格搭配柱状、饼图可视化展示平台交易活跃度、各院系闲置流通情况。

### 7. 校园学生小程序端
师生通过微信小程序浏览闲置商品、发布出售/求购信息；查看个人全部订单与历史交易记录。

## 二、课程设计目的

《前端综合实践》课程设计是软件工程专业本科《前端设计与开发》、《数据库原理》、《软件工程》和《软件质量与测试》课程配套实践环节，是课程教学不可或缺的核心实践内容。本课程设计旨在让学生围绕小型软件工程项目，完整掌握软件项目管理、需求分析、系统设计、编码和测试等全流程标准方法与技术；帮助学生深度理解软件开发模型、软件生命周期、标准化软件过程等理论知识在实际项目落地中的应用价值。

通过实践训练，达成以下培养目标：
- 掌握遵循软件工程规范、标准、方法开展软件开发的综合实践能力；
- 培养小组协作开发意识、项目分工与团队沟通能力；
- 熟练撰写标准化软件技术文档，提升专业文档撰写能力；
- 建立软件项目整体管控思维，提升小型软件项目统筹管理能力。

## 四、课程设计要求

课程设计选用真实场景业务系统作为开发题目，以 5 人小组为单位独立完成全部设计和开发工作，要求学生将软件工程理论方法落地到真实系统设计中，完整输出可行性分析、需求分析、系统概要设计、详细设计、测试用例、部署手册全套技术文档。

- **分组规则**：学生自由组队，每组人数 5 人；组员明确分工，组长统筹协调，协作完成，个人贡献纳入考核评价，有“搭便车”行为或不完成分配任务将扣分及至判为不及格。
- **设计周期**：4 周，设计验收时机房现场进行系统演示和答辩，并提交纸质版完整报告，电子版由班长统一收集后发送至任课教师邮箱。
- **文档规范**：严格参照课程设计教材各阶段标准任务与文档模板完成全套设计文档，项目各模块工作需明确划分责任人，并在文档中标注对应完成人员。
- **考核纪律**：各组独立完成全部设计内容，报告存在抄袭、组间雷同、大段网络原文复制等情况，本次课程设计成绩统一记 0 分。
- **免重复提交政策**：参与计算机设计大赛、软件开发类竞赛且已完整产出需求分析、系统设计和实现全套文档的学生，可将原有项目文档按照本课程设计报告格式重新整理排版后提交；参赛学生需提前向教师报备班级、学号、姓名、参赛赛事名称及获奖名次。

## 五、提交报告文档及要求

| 序号 | 报告名称    | 目的要求                                                                               |
|----|---------|------------------------------------------------------------------------------------|
| 1  | 可行性分析报告 | 采用系统化、全维度分析思路，以项目经济可行性为核心，综合梳理项目技术、操作、法律、市场等各类影响因素，依托数据、业务资料论证项目落地可行性，输出完整可行性分析报告。 |
| 2  | 需求规格说明书 | 结合具体业务场景完成全维度需求调研与梳理，规范绘制分层数据流图、完整数据字典、实体联系 ER 图，最终形成标准化需求规格说明书。                   |
| 3  | 概要设计说明书 | 搭建系统顶层软件架构结构图，完成模块拆分、耦合优化、分层改进，输出系统概要设计说明书。                                        |
| 4  | 详细设计说明书 | 完成各功能模块人机交互界面设计，使用程序流程图、盒图等标准过程设计工具描述模块内部执行逻辑，输出详细设计说明书。                           |
| 5  | 源代码     | 完整提交项目全部源码文件，包含前端页面、后端业务逻辑、数据库脚本、配置文件；代码规范整洁，添加关键注释，分模块整理归档，附带代码目录说明，保证可正常编译运行。    |
| 6  | 测试用例文档  | 围绕系统核心业务功能设计功能测试用例，包含测试模块、测试场景、输入数据、预期结果、实际结果、测试结论；覆盖正常流程、异常边界、权限校验等场景，配套简单测试结果说明。 |
| 7  | 系统部署手册  | 说明系统软硬件运行环境、开发环境依赖、数据库安装配置、项目部署步骤、启动与关闭流程、常见故障排查方案，零基础人员可按照手册完成系统本地部署运行。           |

**补充说明**：全套文档必须配套完整工程图表，包含数据流图、ER 图、数据字典、软件层次结构图、程序流程图、盒图等；同时附上系统界面原型截图；所有源码、测试文档、部署手册统一打包随电子版报告一并提交。
